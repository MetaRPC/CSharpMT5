    Getting Started with CSharpMT5¬∂

    Welcome to CSharpMT5 - a comprehensive educational project for learning MT5 trading automation from the ground up using C# and .NET. This guide will show you the learning path and what opportunities this project opens for you.

    üöÄ Prerequisites and Setup¬∂

    Before you start working with CSharpMT5, you need to set up your development environment.

    Step 1: Install .NET 8 SDK¬∂

    CSharpMT5 requires .NET 8 SDK or higher.

    Download and Install:

    • Official: .NET 8 SDK Download
    • Choose the installer for your platform (Windows, macOS, Linux)

    Verify installation:

    dotnet --version
# Should show: 8.0.x or higher

    Check .NET installation:

    dotnet --info
# Should display SDK version, runtime, and environment info

    No additional setup needed! Unlike Java/Maven, .NET CLI is fully integrated and ready to use immediately after installation.

    Step 2: Install a Code Editor (Optional but Recommended)¬∂

    Visual Studio Code (Recommended):

    • Download: VS Code
    • Extensions to install:
    • C# Dev Kit (official Microsoft extension)
    • C# (OmniSharp)

    Visual Studio 2022 (Alternative):

    • Download: Visual Studio 2022
    • Choose "Community" edition (free)
    • Select ".NET desktop development" workload during installation

    You can also use: - JetBrains Rider - Visual Studio for Mac - Any text editor + command line

    Step 3: Configure MT5 Gateway Connection¬∂

    CSharpMT5 connects to MT5 terminal via gRPC gateway (typically mt5term).

    What is mt5term Gateway?

    • External gateway process that bridges CSharpMT5 to MT5 terminal
    • Handles connection pooling and session management
    • No need to launch MT5 terminal separately when using gateway
    • Gateway must be running before connecting

    Connection flow: 

    CSharpMT5 ‚Üí gRPC ‚Üí mt5term Gateway ‚Üí MT5 Terminal

    Two ways to connect:

    Option A: Connect to Local MT5 Terminal via Gateway¬∂

    If you have mt5term gateway running locally:

    1. Ensure gateway is running (mt5term executable)
    2. Gateway connects to your MT5 terminal
    3. Configure Config\appsettings.json:
    {
  "MT5Connection": {
    "Host": "localhost",
    "Port": 5555,
    "Login": 12345678,
    "Password": "your_password",
    "Symbol": "EURUSD",
    "UseSSL": false,
    "TimeoutSeconds": 30
  }
}

    Option B: Connect to Remote Gateway (Provided by MetaRPC)¬∂

    If MetaRPC team provides access to remote gateway:

    1. Obtain connection credentials from MetaRPC
    2. Configure Config\appsettings.json with provided settings:
    {
  "MT5Connection": {
    "Host": "gateway.metarpc.com",     // Provided by MetaRPC
    "Port": 5555,                      // Provided by MetaRPC
    "Login": 123456,                   // Your MT5 account number
    "Password": "your_password",       // Your MT5 password
    "Symbol": "EURUSD",
    "UseSSL": true,                    // Usually true for remote
    "TimeoutSeconds": 30
  }
}

    Step 4: Clone and Build the Project¬∂

    Clone repository:

    git clone https://github.com/MetaRPC/CSharpMT5.git
cd CSharpMT5

    Understanding dotnet run - Your Project Command:

    The dotnet run command is the central command for CSharpMT5:

    • Runs examples (market, positions, pendingorders)
    • Runs orchestrators - trading strategies (grid, scalping, hedge, news, breakout)
    • Runs presets - multi-strategy systems (preset, adaptive)
    • Auto-compiles the project before running
    • Restores NuGet packages automatically if needed

    What happens when you run dotnet run market:

    1. dotnet restores NuGet packages (if needed)
2. Compiles all C# source files
3. Launches Program.cs Main()
4. Routes to Program.Sugar.MarketOrders example

    This ensures every run starts with a fresh, complete build.

    For complete command reference:

    All available commands are documented in Program.cs header. Quick examples:

    # Examples (learning API)
dotnet run market          # Sugar market orders
dotnet run positions       # Service positions demo
dotnet run pendingorders   # Sugar pending orders

# Orchestrators (strategies)
dotnet run grid            # Grid trading
dotnet run scalping        # Scalping strategy
dotnet run hedge           # Hedging strategy

# Presets (multi-strategy)
dotnet run preset          # Adaptive preset

    First build:

    # Windows PowerShell, CMD, or any terminal
dotnet run market

# This will:
# 1. Restore NuGet packages (Grpc.Net.Client, Google.Protobuf, etc.)
# 2. Compile the project
# 3. Run Sugar market orders example

    If you get errors:

    Common fixes: 

    # Clean and rebuild
dotnet clean
dotnet build

# Restore packages explicitly
dotnet restore

# Check .NET version
dotnet --version   # Should be 8.0 or higher

    Step 5: Verify Everything Works¬∂

    Test the setup:

    # Run a simple example
dotnet run market

# If it connects and runs - you're ready!
# If you see errors - check connection settings

    üéØ What is CSharpMT5?¬∂

    CSharpMT5 is an educational project designed specifically to teach you how to work with MetaTrader 5 terminal at all levels - from low-level protocol communication to high-level trading strategies using C# and .NET.

    This project was initially created to learn low-level methods of MT5 terminal communication. Everything else - convenience layers, orchestrators, presets - was built along the way to make this knowledge more accessible and practical.

    Why C#? - Modern language with excellent async/await support - Strong typing and great IDE support - Cross-platform (.NET runs on Windows, macOS, Linux) - Powerful ecosystem (NuGet, gRPC libraries, etc.) - Extension methods for clean API design

    üéì What You'll Learn¬∂

    1. Low-Level Protocol Communication (Foundation)¬∂

    The core foundation of everything - direct communication with MT5 terminal via gRPC protocol.

    You'll learn:

    • How MT5 terminal communicates via Protocol Buffers (protobuf)
    • How to send/receive proto Request/Response objects
    • Direct gRPC calls for all MT5 operations
    • Connection management with ExecuteWithReconnect retry logic
    • Raw access to every MT5 function
    • Async/await patterns for non-blocking I/O

    API: MT5Account.cs (Layer 1)

    Documentation: MT5Account.Master.Overview.md

    Why this matters: Understanding the low-level foundation gives you complete control and deep knowledge of how everything works under the hood.

    2. Wrapper Layer (Simplification)¬∂

    Built on top of MT5Account - simplified method signatures without proto complexity.

    You'll learn:

    • How to wrap proto objects into simple method calls
    • Type conversions (proto ‚Üí C# primitives)
    • Extension methods for API design
    • Simplified API design patterns
    • Direct data returns without proto wrapping

    API: MT5Service.cs + MT5Service.Sugar.cs (Layer 2)

    Why this matters: Shows how to build convenient APIs on top of complex protocols.

    3. Convenience Layer (Sugar Methods)¬∂

    Built on top of MT5Service - ready-to-use methods with smart features.

    You'll learn:

    • Auto-normalization of volumes and prices
    • Risk-based position sizing (calculate volume from $ risk)
    • Points-based methods (BuyLimitPoints, SellStopPoints, etc.)
    • Batch operations (CloseAll, CancelAll)
    • Smart helpers (GetAccountSnapshot, GetSymbolSnapshot)
    • Retry logic with exponential backoff

    API: MT5Sugar.cs (Layer 3)

    Documentation: Planned in MT5Sugar/

    Why this matters: Shows how to build production-ready convenience APIs that handle edge cases automatically.

    4. Trading Strategy Implementation (Orchestrators)¬∂

    Built using MT5Sugar - complete trading strategy workflows.

    You'll learn:

    • How to implement real trading strategies in code
    • Risk management and position sizing
    • Position monitoring with progress bars
    • Entry/exit automation
    • Performance tracking (balance, equity, P/L)
    • Cancellation handling (Ctrl+C)

    Strategies included:

    • Grid Trading - orders at regular intervals (range-bound markets)
    • Simple Scalping - quick trades with tight stops
    • Quick Hedge - hedging strategy (high volatility)
    • News Straddle - breakout trading around news events
    • Pending Breakout - pending orders for breakouts

    Location: Examples\Orchestrators\

    Documentation: Strategies.Master.Overview.md

    Why this matters: Real-world strategy implementation patterns you can adapt for your own trading ideas.

    5. Multi-Strategy Systems (Presets)¬∂

    Combine multiple orchestrators - adaptive trading systems with conditional logic.

    You'll learn:

    • How to combine multiple strategies
    • Adaptive decision-making based on market analysis
    • Volatility calculation and condition detection
    • Multi-phase trading sessions
    • Strategy orchestration patterns
    • Performance tracking across strategies

    Presets included:

    • Adaptive Market Mode - Intelligent system that analyzes market (volatility, time, news) and selects optimal orchestrator

    Location: Examples\Presets\

    Documentation: AdaptiveMarketModePreset.md

    Why this matters: Advanced strategy composition - how to build complex trading systems from simple building blocks.
