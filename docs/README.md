> Черновик сгенерирован 2025-08-30 15:03:33.

Этот скелет поможет тебе быстро довести документацию до состояния «как в GoMT4 Skeleton», но с учётом **шорткастов** и особенностей MT5.

## Как пользоваться этим скелетом

1. Открой раздел **Шорткасты** — там базовое объяснение концепции и шаблоны.
2. Перейди в **API Reference** — по каждой публичной `class` создана страница со списком методов.
3. В конце работы мы просто «распихаем» страницы по уже настроенной системе навигации. Ничего не ломаем.

> Важно: «дохлые методы» (неиспользуемые) **специально не включены** в список. Для твоего удобства я сгенерировал CSV-таблицу `methods_index.csv`, где ты сможешь быстро отметить «dead/keep» и мы это отфильтруем при следующей итерации.

## Quick Start

```csharp
// Minimal example (pseudo):
// 1) Create client
var client = new Mt5Client(/* config */);

// 2) Connect
await client.ConnectAsync();

// 3) Do a quote or order
var quote = await client.GetQuoteAsync("EURUSD");
Console.WriteLine($"Bid: {quote.Bid}, Ask: {quote.Ask}");

// 4) Disconnect
await client.DisconnectAsync();
```

## Шорткасты (что это и зачем)

Шорткасты — это «короткие вызовы»/«сахар» поверх стандартного API, чтобы писать меньше кода и избегать рутины. 
Они **не заменяют** базовые RPC/методы, а **дополняют** их удобными обёртками.

- Кому полезно: новичкам, прототипам, скриптам «на скорую руку».
- Как устроено: каждый шорткаст — это обычный метод/объект, который внутри вызывает несколько стандартных операций.

Подробнее см. `docs/shortcasts/Overview.md`.
